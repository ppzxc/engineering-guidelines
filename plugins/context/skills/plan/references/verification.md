# Verification Gates for context:plan

context:plan 각 산출 스텝에 적용되는 tier 1/2 검증 게이트 정의.
SKILL.md에서 각 스텝 말미에 "이 스텝 완료 후 이 파일의 해당 게이트를 실행하라"로 참조된다.

---

## Tier 정의

### Tier 1 — Self-Review Gate

`superpowers:verification-before-completion` Skill을 명시 호출한 후
아래 스텝별 체크리스트를 실행한다. "evidence before claims" 원칙:
각 항목을 명령 출력·파일 내용으로 증거 확정 후 통과 표기.

실패 처리:
- **자동 수정** 가능 항목: TBD/placeholder 제거, 체크박스 수 불일치 재추출
- **User Gate** 필요 항목: 섹션 간 모순, 이중 해석 가능 요구사항

### Tier 2 — GAN Cross-check

plan 전문을 비-Claude CLI에 전달한다. 리뷰어는 적대적 discriminator 역할.
H-severity 결함 발견 시 **AI(context:plan을 실행 중인 Claude)**가 plan.md 본문을 1회 직접 편집한다.
편집 후 미해결 H-severity → context.md Blockers 이관. 재실행·User Gate 없음.

## Step-4 Gate (spec.md)

Invoke Skill: `superpowers:verification-before-completion`

Tier 1 체크리스트:
- [ ] "TBD", "TODO", 불완전 섹션 없음 → 있으면 자동 수정
- [ ] 섹션 간 내부 모순 없음 → 있으면 User Gate
- [ ] 단일 구현 플랜으로 실행 가능한 범위 → 모호하면 User Gate
- [ ] 이중 해석 가능 요구사항 없음 → 있으면 User Gate (하나로 고정)
- [ ] `test -f docs/context/{TASK_NAME}/spec.md` → exit 0

## Step-6 Gate (plan.md)

실행 순서 (이 순서를 반드시 지킬 것):
1. `test -f docs/context/{TASK_NAME}/plan.md` → exit 0
2. Tier 2 GAN cross-check (아래 CLI 섹션 참조)
3. H-severity 항목 → plan.md 본문 직접 편집 (AI 수행, 1회 한정)
4. Invoke Skill: `superpowers:verification-before-completion`
5. Tier 1 체크리스트 실행 (편집 결과 포함하여 재검토)

Tier 1 체크리스트:
- [ ] `test -f docs/context/{TASK_NAME}/plan.md` → exit 0
- [ ] Goal·Architecture·Tech Stack 섹션 존재
- [ ] File Structure 섹션 존재
- [ ] plan.md에 `- [ ]` 태스크 목록 존재 (추출 가능)
- [ ] Tier 2 실행됨 (provenance 라인 준비됨)

---

## Step-7 Gate (tasks.md, context.md)

실행 순서 (이 순서를 반드시 지킬 것):
1. `ORIG_COUNT=$(grep -c -- '- \[ \]' docs/context/{TASK_NAME}/plan.md)` — 트림 전 캡처
2. tasks.md 추출 (Phase별 그룹·`[P]` 병렬 마커·체크포인트 보존)
3. Invoke Skill: `superpowers:verification-before-completion`
4. Tier 1 체크리스트 실행
5. context.md 생성
6. plan.md 트림 (`### Task N` 블록 제거)
7. 트림 후 검증: `grep -c '### Task' docs/context/{TASK_NAME}/plan.md` == 0

Tier 1 체크리스트:
- [ ] tasks.md의 `- [ ]` 수 == ORIG_COUNT (무손실)
- [ ] Phase 그룹·`[P]` 병렬 마커·체크포인트 보존됨
- [ ] `context.md` Decision Log: 내용 있음 (grill 결정 캡처됨)
- [ ] `context.md` Key Files: 내용 있음
- [ ] `context.md`에 `<!-- last_updated: ... -->` 라인 존재

## Tier 2 CLI 실행 패턴

### 우선순위
AGY CLI → Gemini CLI → Codex CLI → Claude self-generate

### GAN-mode 프롬프트 템플릿

```
You are an adversarial discriminator. The plan below was generated by a different model.
Your mission: identify ALL defects that would cause failure during implementation.

Attack assumptions, expose omissions, flag ordering risks, and call out anything infeasible.
Do NOT validate or praise. Assume there are defects until proven otherwise.

Output format:
1. Cross-check table: | tag | finding | severity(H/M/L) |
   tags: consistency, omission, ordering, risk, feasibility, version-compat
2. Test Scenarios (3): normal case, boundary case, failure case — with how to verify
3. Pre-mortem (3): specific scenarios where this plan fails in production

GAN MODE ACTIVE. Begin.

--- PLAN START ---
{PLAN_CONTENT}
--- PLAN END ---
```

### 호스트 감지 (ADR-0022/0023 — 자기 자신 cross-check 금지)
- Claude Code 호스트 → AGY/Gemini/Codex CLI 위임
- Gemini CLI 호스트 → AGY/Codex CLI 우선, `claude -p` fallback

### CLI 호출 (입력은 stdin/임시파일로만, 인자 보간 금지)

**Pre-flight:**
```bash
timeout 3 agy --version    || # AGY_NOT_FOUND: 발동 → Gemini로
timeout 3 gemini --version || # CLI_NOT_FOUND: 발동 → Codex로
timeout 3 codex --version  || # CLI_NOT_FOUND: 발동 → self-generate로
```

**AGY (우선):**
```bash

test -f "docs/context/${TASK_NAME}/plan.md" || { echo "CLI_ERROR: plan.md not found"; exit 1; }
PLAN_CONTENT=$(cat "docs/context/${TASK_NAME}/plan.md")
TMPFILE=$(mktemp)
trap 'rm -f "$TMPFILE"' EXIT
# Note: agy -p "$(cat $TMPFILE)" passes content as arg; safe for plans <100KB
cat > "$TMPFILE" << 'GANEOF'
You are an adversarial discriminator. The plan below was generated by a different model.
Your mission: identify ALL defects that would cause failure during implementation.

Attack assumptions, expose omissions, flag ordering risks, and call out anything infeasible.
Do NOT validate or praise. Assume there are defects until proven otherwise.

Output format:
1. Cross-check table: | tag | finding | severity(H/M/L) |
   tags: consistency, omission, ordering, risk, feasibility, version-compat
2. Test Scenarios (3): normal case, boundary case, failure case — with how to verify
3. Pre-mortem (3): specific scenarios where this plan fails in production

GAN MODE ACTIVE. Begin.

--- PLAN START ---
GANEOF
printf '%s\n--- PLAN END ---\n' "$PLAN_CONTENT" >> "$TMPFILE"
timeout 330 agy -p "$(cat "$TMPFILE")" --print-timeout 300

```

### Sentinel 처리 (동일 인자 재호출 금지)

| Sentinel prefix | 의미 | 처리 |
|----------------|------|------|
| `AGY_NOT_FOUND:` | agy 부재 | Gemini로 |
| `AGY_TIMEOUT:` | 300s 초과 | Gemini로 |
| `AGY_ERROR(exit=N):` | 비정상 종료 | Gemini로 |
| `CLI_NOT_FOUND:` | gemini/codex 부재 | 다음 CLI로 |
| `CLI_TIMEOUT:` | 300s 초과 | 다음 CLI로 |
| `CLI_ERROR(exit=N):` | 비정상 종료 | 다음 CLI로 |
| (모두 실패) | — | Claude self-generate |

Conservative fallback: 동일 스키마 출력, 리뷰어 = "Claude self-generate".
