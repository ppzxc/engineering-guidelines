# Verification Gates for context:plan

context:plan Step 6에 적용되는 Tier 2 GAN 검증 게이트.

**게이트 설계 (2개):**
- spec → 사람 리뷰 (AskUserQuestion, Step 5) [의도 검증 — 최우선]
- plan → GAN cross-check (Step 6, 플랜모드 내 Opus) [실행가능성 검증]

self-review(Tier 1) 게이트는 제거됨. 근거: 모델이 자기 출력을 자기검증하는 것은 약하며,
사람(spec) + 다른 모델(plan)의 적대적 리뷰가 각기 다른 실패모드를 커버해 self-review와 중복된다.

---

## Step-6 Gate (플랜 파일 내 plan 내용)

> **중요**: Step 6은 플랜모드 내(Opus)에서 실행된다. `docs/context/{TASK_NAME}/plan.md`는 Step 8 이전에 존재하지 않는다. GAN 검증 대상은 **플랜 파일에 기록된 plan 섹션 내용**이다. TMPFILE은 플랜 파일에서 plan 내용을 추출하여 빌드한다.

실행 순서 (이 순서를 반드시 지킬 것):

1. 플랜 파일(플랜모드 파일)에서 plan 섹션 내용 추출 → TMPFILE 빌드
2. Tier 2 GAN cross-check (아래 CLI 섹션 참조)
3. H-severity 항목 → **플랜 파일** plan 섹션 직접 보정 (Opus 수행, 1회 한정)
4. 미해결 H-severity → Step 8 context.md Blockers 이관

## Tier 2 CLI 실행 패턴

### 우선순위
AGY CLI → Gemini CLI → Codex CLI → Claude self-generate

### 호스트 감지 (ADR-0022/0023 — 자기 자신 cross-check 금지)
- Claude Code 호스트 → AGY/Gemini/Codex CLI 위임
- Gemini CLI 호스트 → AGY/Codex CLI 우선, `claude -p` fallback

### CLI 호출 (입력은 stdin/임시파일로만, 인자 보간 금지)

**Pre-flight:**
```bash
timeout 3 agy --version    2>/dev/null || echo "AGY_NOT_FOUND:"
timeout 3 gemini --version 2>/dev/null || echo "CLI_NOT_FOUND:gemini"
timeout 3 codex --version  2>/dev/null || echo "CLI_NOT_FOUND:codex"
```

**TMPFILE 빌드 (세 CLI 공통 — 호출 전 실행):**
```bash
# PLAN_CONTENT = 플랜 파일(플랜모드 파일)에서 plan 섹션 내용 추출
# (docs/context/{TASK_NAME}/plan.md는 아직 존재하지 않음 — Step 8에서 생성됨)
# 오케스트레이터가 플랜 파일 내 plan 내용을 문자열로 전달한다.
PLAN_CONTENT="<플랜 파일에서 추출한 plan 섹션 내용>"
TMPFILE=$(mktemp)
trap 'rm -f "$TMPFILE"' EXIT
cat > "$TMPFILE" << 'GANEOF'
You are an adversarial discriminator. The plan below was generated by a different model.
Your mission: identify ALL defects that would cause failure during implementation.

Attack assumptions, expose omissions, flag ordering risks, and call out anything infeasible.
Do NOT validate or praise. Assume there are defects until proven otherwise.

Output format:
1. Cross-check table: | tag | finding | severity(H/M/L) |
   tags: consistency, omission, ordering, risk, feasibility, version-compat, discipline
2. Test Scenarios (3): normal case, boundary case, failure case — with how to verify
3. Pre-mortem (3): specific scenarios where this plan fails in production

Additionally evaluate discipline injection in plan.md:
- Karpathy lens reflected in scope (no speculative features, surgical changes only)?
- Plan.md task plan intends [S]/[B] tagging and [S]-first ordering per phase?
- Non-trivial behavioral tasks intended to decompose into RED→GREEN→REFACTOR?
Note: final tasks.md is validated mechanically in Step 8; here, only the plan-level intent.
Flag violations severity H only if discipline is enabled (see DISCIPLINE token above).

DISCIPLINE: {DISCIPLINE_STATE}

GAN MODE ACTIVE. Begin.

--- PLAN START ---
GANEOF
printf '%s\n--- PLAN END ---\n' "$PLAN_CONTENT" >> "$TMPFILE"
```

> `{DISCIPLINE_STATE}` 자리에 `karpathy=on tdd-tidy=on` 또는 `karpathy=off(<reason>) tdd-tidy=on` 형태로 실제 상태를 주입한다.
> `NO_KARPATHY` / `NO_TDD_TIDY` 미설정 → `on`, 설정 → `off(<reason>)`.
> discriminator는 이 토큰으로 opt-out 여부를 판정한다 (spec.md blockquote 불필요).

**AGY (우선):**
```bash
printf '%s' "$(cat "$TMPFILE")" | timeout 330 agy --print-timeout 300s
```

**Gemini (fallback):**
```bash
printf '%s' "$(cat "$TMPFILE")" | timeout 300 gemini -p "GAN MODE ACTIVE — see stdin for plan"
```

**Codex (fallback):**
```bash
cat "$TMPFILE" | timeout 300 codex exec -
```

### Sentinel 처리 (동일 인자 재호출 금지)

| Sentinel prefix | 의미 | 처리 |
|----------------|------|------|
| `AGY_NOT_FOUND:` | agy 부재 | Gemini로 |
| `AGY_TIMEOUT:` | 300s 초과 | Gemini로 |
| `AGY_ERROR(exit=N):` | 비정상 종료 | Gemini로 |
| `CLI_NOT_FOUND:` | gemini/codex 부재 | 다음 CLI로 |
| `CLI_TIMEOUT:` | 300s 초과 | 다음 CLI로 |
| `CLI_ERROR(exit=N):` | 비정상 종료 | 다음 CLI로 |
| (모두 실패) | — | Claude self-generate |

Conservative fallback: 동일 스키마 출력, 리뷰어 = "Claude self-generate".
