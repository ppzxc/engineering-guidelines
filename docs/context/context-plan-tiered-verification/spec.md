# context:plan 스텝별 계층형 자가검증·리뷰 게이트

_설계 확정: 2026-05-27 (grill-me 검증 완료)_

---

## Context and Problem Statement

`context:plan`은 raw 아이디어 → 4파일 Dev Docs 폴더를 7스텝으로 오케스트레이션한다.
실제 스킬 코드 확인 결과 검증 분포가 불균형하다:

| 스텝 | 산출물 | 현재 검증 |
|------|--------|-----------|
| 4 brainstorming | spec.md | 두텁다 — brainstorm self-review + grill-me + User Gate + 파일위치 검증 |
| 6 writing-plans | plan.md | **얇다** — `test -f plan.md` 존재 확인뿐, 내용 리뷰 없음 |
| 7 추출/합성 | tasks.md, context.md | **없다** — 기계적 추출만, 무결성 검증 전무 |

방법론(`superpowers:verification-before-completion`, `git:review`의 peer cross-check 패턴)은
프로젝트에 이미 있으나 context:plan에 연결돼 있지 않다.
자매 스킬 `workflow:planning`은 plan에 멀티 LLM 교차검증을 걸지만
context:plan은 같은 plan 산출물을 만들면서 그 게이트가 없다.

---

## Goals

- 각 산출 스텝(4/6/7)에 계층형 검증 게이트를 추가한다
- context 플러그인 안에 self-contained로 구현한다 (외부 리뷰 스킬 신규 의존 없음)

## Non-Goals

- `workflow` 플러그인 변경 없음
- spec/plan 산출 경로 변경 없음 (ADR-0027 경계 유지)
- 기존 brainstorming/grill-me/writing-plans 파이프라인 교체 없음

---

## Constraints (확정)

1. `superpowers:verification-before-completion` Skill 도구 호출만 허용.
   `workflow:planning`·`llm`·`agy` 스킬 추가 호출 금지.
   기존 파이프라인 의존(brainstorming/grill-me/writing-plans) 유지.
2. 멀티 LLM은 AGY/Gemini/Codex **CLI 직접 호출**만.
3. tier 2: 자동 + graceful fallback + 리뷰모델 provenance 로깅 + GAN-mode 프롬프트.

---

## Design

### tier 정의

**tier 1 — Self-Review Gate** (저비용, 추가 LLM 없음)

산출물을 스텝별 체크리스트로 자가 재검토한다.
`superpowers:verification-before-completion` Skill을 명시 호출하여 "evidence before claims" 원칙을 적용한다:
각 체크리스트 항목을 명령 출력·파일 내용으로 증거 확정 후 통과 표기.

실패 처리 (D안 확정):
- **자동 수정** 가능한 항목: TBD/placeholder 제거, 체크박스 수 불일치 재추출
- **User Gate** 필요 항목: 섹션 간 모순, 이중 해석 가능한 요구사항

**tier 2 — GAN Cross-check** (멀티 LLM CLI)

plan 전문을 비-Claude CLI에 stdin pipe로 전달한다.
리뷰어 LLM은 **적대적 discriminator** 역할: 결함 적발이 임무, 칭찬·검증 금지.

H-severity 결함 처리 (B안 + A안 확정):
- Claude가 plan.md 본문을 **1회 직접 편집**하여 수정 시도
- 미해결 H-severity → context.md Blockers에 이관
- 재실행 없음, User Gate 없음

출력 스키마:
- Cross-check 표: `tag | item | severity(H/M/L)` — tags: consistency/omission/ordering/risk/feasibility/version-compat
- Test Scenarios × 3 (정상·경계·실패)
- Pre-mortem × 3 (구체적 실패 시나리오)

### 스텝 → tier 매핑

| 스텝 | 산출물 | 추가 게이트 |
|------|--------|-------------|
| 4 | spec.md | **tier 1** — TBD/placeholder 스캔, 내부 일관성, 정본 경로 확정 |
| 6 | plan.md | **tier 2 → 수정 → tier 1** (이 순서 고정) |
| 7 | tasks.md, context.md | **tier 1** — 추출 무결성 검증 |

### Step 6 내부 실행 순서 (A안 확정)

```
1. test -f plan.md (파일위치 검증)
2. tier 2 GAN cross-check (AGY CLI 우선)
3. H-severity 항목 → plan.md 본문 직접 편집
4. Invoke Skill: superpowers:verification-before-completion
5. tier 1 체크리스트 실행
6. Step 7로 진행
```

### Step 7 내부 실행 순서 (A안 확정, 타이밍 버그 픽스)

```
1. ORIG_COUNT=$(grep -c '- \[ \]' plan.md)   # 트림 전 카운트 캡처
2. tasks.md 추출 (Phase별 그룹·[P]·체크포인트 보존)
3. Invoke Skill: superpowers:verification-before-completion
4. tier 1 체크리스트 실행
   - tasks.md - [ ] 수 == ORIG_COUNT (무손실)
   - Phase·[P]·체크포인트 보존 여부
   - context.md Decision Log 내용 있음
   - context.md Key Files 내용 있음
   - last_updated 라인 존재
5. context.md 생성
6. plan.md 트림 (### Task N 블록 제거)
7. 트림 후 검증: grep -c '### Task' plan.md == 0
```

### tier 1 체크리스트 — 스텝별

**Step 4 (spec.md)**:
- [ ] "TBD", "TODO", 불완전 섹션 없음 (있으면 자동 수정)
- [ ] 섹션 간 내부 모순 없음 (있으면 User Gate)
- [ ] 단일 구현 플랜으로 실행 가능한 범위 (모호하면 User Gate)
- [ ] 이중 해석 가능한 요구사항 없음 (있으면 User Gate → 하나로 고정)
- [ ] `test -f docs/context/{TASK}/spec.md` exit 0

**Step 6 (plan.md)**:
- [ ] `test -f docs/context/{TASK}/plan.md` exit 0
- [ ] Goal·Architecture·Tech Stack 섹션 존재
- [ ] File Structure 섹션 존재
- [ ] `- [ ]` 태스크 목록 존재 (추출 가능)
- [ ] tier 2 cross-check 실행됨 (또는 fallback 기록됨)

**Step 7 (tasks.md, context.md)**:
- [ ] `tasks.md` - [ ] 수 == ORIG_COUNT (무손실)
- [ ] Phase 그룹·`[P]` 병렬 마커·체크포인트 보존됨
- [ ] `context.md` Decision Log: 내용 있음 (grill 결정 캡처됨)
- [ ] `context.md` Key Files: 내용 있음 (plan 추출됨)
- [ ] `context.md` `<!-- last_updated: ... -->` 라인 존재
- [ ] `plan.md` 트림 후: `### Task N` 블록 없음

### GAN-mode 프롬프트 (tier 2, CLI 전달용)

```
You are an adversarial discriminator. The plan below was generated by a different model.
Your mission: identify ALL defects that would cause failure during implementation.

Attack assumptions, expose omissions, flag ordering risks, and call out anything infeasible.
Do NOT validate or praise. Assume there are defects until proven otherwise.

Output format:
1. Cross-check table: | tag | finding | severity(H/M/L) |
   tags: consistency, omission, ordering, risk, feasibility, version-compat
2. Test Scenarios (3): normal case, boundary case, failure case — with how to verify
3. Pre-mortem (3): specific scenarios where this plan fails in production

GAN MODE ACTIVE. Begin.

--- PLAN START ---
{PLAN_CONTENT}
--- PLAN END ---
```

### CLI 호출 · 안전 · Fallback (확정)

**우선순위**: AGY CLI → Gemini CLI → Codex CLI → Claude self-generate

**호스트 감지 및 자기 cross-check 금지 (ADR-0022/0023)**:
- Claude Code 호스트 → AGY/Gemini/Codex CLI 위임 (자기 자신 호출 금지)
- Gemini CLI 호스트 → AGY/Codex CLI 우선, `claude -p` fallback

**입력 전달**: stdin pipe 또는 임시 파일. CLI 인자 직접 보간 금지 (shell injection·ARG_MAX 방지).

```bash
# Pre-flight (세 CLI 공통)
timeout 3 agy --version   || { echo "AGY_NOT_FOUND:"; fallback to gemini }
timeout 3 gemini --version || { echo "CLI_NOT_FOUND:"; fallback to codex }
timeout 3 codex --version  || { echo "CLI_NOT_FOUND:"; fallback to self-generate }

# AGY 호출
TMPFILE=$(mktemp)
printf '%s' "$GAN_PROMPT" > "$TMPFILE"
timeout 330 agy -p "$(cat "$TMPFILE")" --print-timeout 300
rm "$TMPFILE"

# Gemini 호출 (fallback)
printf '%s' "$GAN_PROMPT" | timeout 300 gemini -p "GAN MODE ACTIVE — see stdin for plan"

# Codex 호출 (fallback)
printf '%s' "$GAN_PROMPT" | timeout 300 codex exec -
```

**Sentinel 처리** (동일 인자 재호출 금지):

| Sentinel prefix | 의미 | 처리 |
|-----------------|------|------|
| `AGY_NOT_FOUND:` | agy 부재 | 다음 CLI로 |
| `AGY_TIMEOUT:` | 300s 초과 | 다음 CLI로 |
| `AGY_ERROR(exit=N):` | 비정상 종료 | 다음 CLI로 |
| `CLI_NOT_FOUND:` | gemini/codex 부재 | 다음 CLI로 |
| `CLI_TIMEOUT:` | 300s 초과 | 다음 CLI로 |
| `CLI_ERROR(exit=N):` | 비정상 종료 | 다음 CLI로 |
| (모두 실패) | — | Claude self-generate (Conservative) |

Conservative fallback: 동일 스키마 출력, 리뷰어 = "Claude self-generate".

### Provenance 로깅
